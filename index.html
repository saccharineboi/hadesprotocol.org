Programmable Blind Arbitration (PBA)

HADES enables adversarial machines to act as arbiters, ignorant of the data they arbitrate[cite: 55]. Traditional models trust servers with decrypted data[cite: 47]. HADES drops this trust assumption; data remains encrypted even when processed by the adversary (arbiter)[cite: 52, 53]. The arbiter gains advantage by processing encrypted data types, not by collecting plaintext[cite: 54].

Core Cryptography

Built on TLS[cite: 7], HADES employs a hybrid cryptographic suite:

  • Key Exchange: X25519 + Crystals-Kyber-1024 [cite: 4, 99]
  • Signatures: Ed25519 + Crystals-Dilithium5 [cite: 4, 99]
  • Tunnel Encryption: ChaCha20-Poly1305 (XChaCha20 variant used [cite: 99]) with keys derived via HKDF-SHA256 [cite: 4, 99]
  • Encrypted Computation: Fully Homomorphic Encryption (FHE - CKKS scheme, full RNS variant) [cite: 5, 99]

This provides classical and post-quantum security foundations.

Mechanism

HADES uses distinct state structures and a specialized data format:

  • Local Programmable States (LPS): Represent individually owned data (e.g., avatars). Owned by a Virtual Identity (VID)[cite: 85]. State changes are made by the owner and broadcast (encrypted) via the adversary[cite: 87, 137]. Contents hidden from the adversary[cite: 88, 138]. Uses AEAD for encryption[cite: 437].
  • Shared Programmable States (SPS): Represent collectively owned data within a session (e.g., game state, virtual world elements)[cite: 166]. Modifications involve a Hadean function (H) executed by the adversary on encrypted data[cite: 71, 167, 196]. Uses FHE for computations[cite: 437]. NULL SPS acts as an identity function[cite: 172, 194].
  • Hadean Transmission Format (HTF): Binary format based on glTF 2.0, augmented for programmable states (.htfl for LPS, .htfs for SPS)[cite: 119, 120]. Includes Lua scripting for behavior[cite: 128, 129].
  • Obols: Ephemeral JSON files used to initiate SPS sessions, containing session parameters, VID public identities, and a passphrase[cite: 230, 231, 247]. Assayed by the adversary[cite: 233].
  • Identities: Manages User Identities (UID), Virtual Identities (VID), and Public Identities (PID)[cite: 104, 113, 116].

Epistemological guarantees ensure uniqueness and explicit transitions between worlds and identities, preventing blending (like AR/AI)[cite: 60, 61, 62, 63]. Decoupling and E2EE address secondary privacy problems[cite: 64, 67].

Proof of Concept: Karpathian Validators (KV)

To demonstrate PBA, KVs were developed for blind validation of encrypted chess moves[cite: 6]. Using FHE (CKKS scheme [cite: 364]), three binary classifiers (MLPs [cite: 362]) differentiate illegal, legal, and checkmate positions with >90% accuracy on encrypted data[cite: 6, 350].

  • Challenge: Validating moves without seeing the board state[cite: 329, 330].
  • Solution: Train models (KVs) on move representations; execute inference homomorphically[cite: 338, 360].
  • Performance: FHE inference is significantly slower (~1M times) and requires large data transfers compared to cleartext[cite: 384, 415, 417]. Accuracy is slightly reduced under FHE[cite: 379, 381].
  • Limitations: KVs exhibit failure modes like occlusion errors, incomplete moves, wormholes, and friendly fire[cite: 407, 408, 409, 410]. These are implementation issues, not protocol flaws[cite: 414].
